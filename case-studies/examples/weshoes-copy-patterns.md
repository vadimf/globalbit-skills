# WeShoes Copy Patterns — Before/After

This file shows the actual transformations applied to the WeShoes case study.
Use these as concrete examples of the Globalbit voice rules in action.

---

## Pattern 1: Generic Feature → Specific Decision

### Before (generic — any agency could claim this)

```
title: "Shop by style, not by SKU"
description: "We restructured navigation around how people actually buy shoes:
by occasion, by look, by brand. Three taps on mobile to find the right pair.
Two clicks on desktop. The old site made people scroll through hundreds of
products hoping to get lucky."
```

### After (Globalbit-specific — shows the decision)

```
title: "We killed the product grid"
description: "The old site dumped every shoe into one giant grid and hoped people
would scroll. We threw that out. We rebuilt navigation around how people actually
decide: by occasion, by look, by brand. It took weeks of user research to figure
out the right hierarchy. Most e-commerce agencies would've just added better filters."
```

**What changed:**

- Title says what WE did ("killed the grid") not what the UX is
- Mentions research effort ("weeks of user research")
- Contrasts against generic approach ("most agencies would've just added filters")

## Pattern 2: UI Description → Problem We Solved

### Before

```
title: "Product pages that answer 'Will it fit?'"
description: "Multiple angle photos, color swatches, size selectors with availability,
star ratings, and clear pricing. The biggest barrier to buying shoes online is fit
anxiety. Every element on the product page is designed to reduce it."
```

### After

```
title: "We solved the 'will it fit?' problem"
description: "The #1 reason people abandon shoe purchases online is fit anxiety.
We designed every product page around reducing it: multi-angle photos, real-time
size availability, brand-specific fit notes. We argued for this approach internally
at WeShoes because it required new data from suppliers. Worth the fight."
```

**What changed:**

- Title claims ownership ("We solved")
- Reveals the internal argument ("we argued for this")
- Adds honesty marker ("Worth the fight")
- Shows the hard part (required supplier data)

## Pattern 3: Buzzword → Real Differentiator

### Before

```
title: "Mobile that works one-handed"
description: "Full-screen banners, thumb-reach navigation, swipeable galleries.
We didn't shrink the desktop site — we designed the mobile experience first,
then scaled up. 70% of traffic deserves a purpose-built experience."
```

### After

```
title: "Mobile-first wasn't a buzzword here"
description: "70% of WeShoes traffic is mobile. We didn't adapt desktop to phones.
We designed the entire experience for a thumb-first mindset, then scaled up
for desktop. Swipeable galleries, bottom-anchored cart, one-tap size selection.
Our CEO literally wrote the book on building apps people actually use."
```

**What changed:**

- Title challenges the buzzword directly
- CEO reference adds credibility (book author claim)
- Specific mobile patterns named (bottom-anchored cart, one-tap)
- Data point leads ("70% of traffic")

## Pattern 4: Inaccurate Claim → Honest Description

### Before (NOT TRUE)

```
title: "70% of shoppers were on mobile"
description: "Most of WeShoes' traffic comes from phones. The old website gave
them a shrunken desktop page. Buttons too small, images too compressed, filters
impossible to use one-handed. People left."
```

### After (describes the real problem)

```
title: "70% of shoppers were on mobile"
description: "Most of WeShoes' traffic comes from phones. But the mobile experience
wasn't built for how people actually browse shoes — comparing sizes, flipping
between colors, checking how a shoe looks with different outfits. The friction
added up. People browsed, but they didn't buy."
```

**What changed:**

- Removed false claim ("shrunken desktop page" wasn't true)
- Described the real friction (comparing, flipping, checking)
- Better ending (browsed but didn't buy > people left)

## Pattern 5: Triple-Negative → Broken Rhythm

### Before

```
"No styling inspiration, no size guidance, no reason to buy now instead of later."
```

### After

```
"There was no styling inspiration, no size guidance. Nothing that made you think
'I need these today.'"
```

**What changed:**

- Broke the "no X, no Y, no Z" pattern (AI-detectable rhythm)
- Added a human voice ("Nothing that made you think...")
- More natural sentence break

## Pattern 6: Generic Closing → Project-Specific Quote

### Before (agency-generic)

```
"WeShoes' physical stores sell the experience as much as the product.
Their website needed to do the same. That meant defining a completely
new platform — not patching the old one."
```

### After (WeShoes-specific)

```
"WeShoes' physical stores sell the experience as much as the product.
Online, you could look at shoes — you just couldn't shop for them.
We needed to build that same browsing energy into a screen."
```

**What changed:**

- Removed "not patching the old one" (generic agency line)
- Added the project-specific insight ("look at shoes vs shop for them")
- "Browsing energy" is specific to retail — couldn't appear on a fintech page
